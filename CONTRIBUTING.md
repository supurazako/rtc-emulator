# Contributing

Thanks for your interest in contributing to `rtc-emulator`.

## How to Contribute

- Use GitHub Issues for bug reports and feature requests.
- Keep proposals focused and small when possible.
- Include enough context so others can reproduce and discuss the change.

## Pull Requests

- Make PRs focused on a single change.
- In your PR description, include:
  - purpose of the change
  - what was changed
  - how you validated it
- Link related issues when applicable (for example: `Closes #123`).
